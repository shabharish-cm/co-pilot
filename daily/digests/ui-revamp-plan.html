Hard constraints & architecture notes
No ANTHROPIC_API_KEY — All Claude calls go through /opt/homebrew/bin/claude --dangerously-skip-permissions -p (already patched in shell/route.ts and claude/route.ts). OAuth session via Claude Pro. Never spawn with ANTHROPIC_API_KEY in env.

CLI is the intelligence layer — The UI is a shell around the CLI, not a replacement. Every smart action (digest, PRD, pulse, notes) goes through a shell spawn. The UI's job is context display, task CRUD, and workflow guidance.

Todoist API — Task CRUD (create, update, complete) calls go directly to Todoist REST API from Next.js server routes — no Claude involved. Board tab uses the existing todoist.ts lib.

Neo-brutalism theme — Retain: dark background (#0d0f14), high-contrast accent colors, 1.5–2px solid borders, monospace for CLI, no rounded-off softness. Add: bold section separators, visible grid lines on Board tab.

State files are the source of truth — current_day.json is the live board state. Digest files are in daily/digests/. Pulse in pulse/. UI reads these; CLI writes them. No database needed.
