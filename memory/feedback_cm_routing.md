Blog posts, landing pages, and marketing content strategy tasks route to **CM**, not Features.

**Why:** CM section is for internal PM execution work. Features is for product development only. Routing content tasks to Features pollutes the feature backlog.

**How to apply:** The full routing table lives in `context/system/routing-and-scoring.md`. This memory is a reminder that "build a landing page" or "write blog posts" are CM tasks even though they contain product-build-sounding language.
