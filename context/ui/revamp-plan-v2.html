01 The actual problem — one diagnosis
What the code does now
Default tab is BOARD. There is no "home" view.
The CLI terminal is a toggleable right sidebar (340px). Closed by default. Requires Cmd+K or clicking CLAUDE button.
DigestPanel lives in a 280px left column on the Board tab only — truncated to 240 chars, full view is a modal.
Board, Pulse, PRD, Settings tabs exist. No dashboard tab.
PRDView is a file browser for existing PRDs — no workflow guidance for creating one.
Terminal and Digest are on different sides of the board, never together.
What the user needs
HOME tab as default: Digest hero + Due Today tasks + CLI always visible.
CLI is a persistent bottom panel on Home — never hidden, no toggle needed.
Full digest as the top zone of Home, not truncated in a sidebar column.
BOARD tab stays — it works. Just not as the landing page.
PRD tab gets a guided multi-step flow: paste input → /research → /jtbd → /prd → /wireframe.
All three zones (Digest, Tasks, CLI) visible at once without toggling anything.
